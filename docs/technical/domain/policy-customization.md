# 정책 커스터마이징과 동적 결재 양식

## 목적

HR 정책은 회사마다 다릅니다. WORKFORCE는 시스템에 회사를 맞추는 방식이 아니라, **회사 정책을 시스템 설정으로 흡수하고 자동화 흐름에 연결하는 방식**을 선택했습니다.

## 커스터마이징 범위

| 영역 | 설정 내용 |
|------|-----------|
| 조직 | 회사별 조직 트리, 부서, 팀, 직급, 직책 |
| 권한 | 리소스, 액션, 범위 기반 역할/권한 |
| 근태 | 출퇴근 슬롯, 연장/야간/휴일 근무 정책, 근무 한도 |
| 휴가 | 회사별 휴가 종류, 부여 기준, 잔여일 관리 |
| 급여 | 급여 항목, 지급/공제, 4대보험, 상여, 퇴직금 정책 |
| 결재 | 결재 양식, 결재선, 참조/공람, 후속 처리 |
| ESG | 활동 유형, 캠페인, 포인트, 교환 상품 |

## JSON 동적 결재 양식

전자결재 양식은 JSON 기반 `formSchema`로 구성합니다.  
새로운 양식이 필요할 때 코드를 수정하지 않고, 관리자가 필드 구조를 등록할 수 있습니다.

```json
{
  "title": "휴가 신청서",
  "fields": [
    { "key": "leaveType", "type": "select", "label": "휴가 종류", "required": true },
    { "key": "startDate", "type": "date", "label": "시작일", "required": true },
    { "key": "endDate", "type": "date", "label": "종료일", "required": true },
    { "key": "reason", "type": "textarea", "label": "사유", "required": false }
  ]
}
```

## AI와 결재 양식 동기화

결재 양식은 화면 렌더링에만 쓰이지 않고 AI 액션과 RAG 검색에도 사용됩니다.

| 연결 지점 | 설명 |
|-----------|------|
| RAG 검색 | 활성 양식이 `approval_form_{documentId}.auto.txt` 문서로 변환되어 Pinecone에 저장 |
| AI 액션 | formSchema가 Function Calling tool로 변환되어 사용자 발화에서 필드값 추출 |
| 조건부 필드 | `visibleWhen` 조건을 고려해 필요한 필드만 추가 질문 |
| 최신성 | 양식 생성/수정/활성화/비활성화 시 `rag.sync.approval` 이벤트 발행 |

## 결재 후속 처리

결재 승인이 끝나면 요청 타입에 따라 후속 처리가 자동으로 실행됩니다.

| 결재 유형 | 후속 처리 |
|-----------|-----------|
| 휴가 신청 | 잔여 연차 차감, 캘린더 등록, 근태 반영 |
| 출퇴근 정정 | 일별 근태 재계산 |
| 외근/출장 | 근태 구분 반영, 캘린더 등록 |
| 인사 발령 | 조직/직책 이력 생성 |
| 계약 | 계약서 상태 변경, PDF 저장 |

## 역할/권한 설계

권한은 리소스, 액션, 범위를 조합해 정의합니다.

| 요소 | 예시 |
|------|------|
| 리소스 | 구성원, 연차, 급여, 결재, 평가 |
| 액션 | READ, CREATE, UPDATE, DELETE |
| 범위 | SELF, TEAM, DEPARTMENT, COMPANY |

이 구조는 발표에서 핵심 기능으로 길게 설명하기보다, AI와 자동화가 안전하게 동작하기 위한 운영 기반 또는 백업 장표로 활용하는 편이 적합합니다.
